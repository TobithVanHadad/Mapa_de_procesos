# Contribución

1. Crear una rama corta desde la rama principal.
2. No incluir secretos, datos personales ni documentos empresariales reales.
3. Mantener reglas de dominio fuera de componentes visuales.
4. Añadir pruebas para cada regla o corrección.
5. Ejecutar `npm run check` antes de solicitar revisión.
6. Documentar migraciones, riesgos y cambios de permisos.

Los cambios de autenticación, autorización, auditoría, migraciones o manejo de archivos requieren revisión de seguridad.
